# ISpooferMotion v2.5.0 (2026-07-16)

## 🎉 Special Thanks

A big thanks to **@gitbannedme / Nikita** for the findings and contributions regarding the global place searching mechanics.

## ✨ New Features

- **Accounts Manager (Accounts View):** A robust new interface to manage multiple Roblox credentials. You can now add accounts using `.ROBLOSECURITY` cookies or OpenCloud API keys. The app automatically verifies credentials, fetches user metadata (Avatar/Name), and lets you assign dedicated roles (Downloader or Uploader). Selecting an account automatically populates your active configuration and disables legacy auto-detect heuristics.
- **Plugin Settings** The Studio Plugin UI now has a Settings tabs.
- **Plugin Connection Status:** Added a visual connection status indicator directly inside the Studio Plugin dock widget, so you always know if the daemon is actively communicating with Studio.

## 🐛 Bug Fixes

- **Global Place Searching:** Replaced naive space-replacement with proper URL percent-encoding. Searches containing symbols like `&`, `=`, or `#` will no longer silently break the backend request.
- **Fast Path Resilience:** The place lookup fast-path (`asset_to_universe_fast_path`) no longer fails silently on HTTP 429 rate limits. It now correctly respects the `Retry-After` header and will retry up to 3 times before falling back to the slower scraping path.
- **Upload Retry Deadlock:** Fixed a state lock in the upload orchestrator where 400-error fallback flags (type/name fallbacks) became permanently locked if the upload hit a 409 Conflict midway. They are now correctly reset during 409 retry iterations.
- **UI Overflow:** Corrected the height calculations in the Results Modal. Extremely large batches of asset replacements will no longer stretch past the viewport on smaller windows.
- **False Positive Replacements:** The UI will no longer display assets as "replaced" if the Studio Bridge injection step fails. State is now strictly synchronized with injection success.
- **Plugin Asset Parsing:** Tightened the asset ID detection heuristic in the Studio plugin. Repeating placeholder numbers (e.g., `00000` or `11111`) are now ignored, and raw numeric strings must meet stricter length requirements before being treated as valid asset IDs.
- **Legacy Log Crashes:** Enforced strict type safety for legacy job history entries. Attempting to view logs for historical jobs that lack a log file will no longer crash the UI.

- **UI Toggle Overflow:** Fixed a visual bug where the thumb of toggle switches would overflow their container bounds when in the "on" position.
- **Missing Translations:** Added missing i18n translation keys.

## 🛠 Internal / Technical

- **Dynamic Plugin Theming:** The Studio Plugin UI now dynamically receives and matches the frontend's Accent Color setting via the daemon's `/health` endpoint.
- **Throttling Engine Overhaul:** Completely rewrote the `AdaptiveLimiter` concurrency mechanism in the Rust backend. Replaced the manual atomic-count + `Notify` loop with a robust, dynamic `tokio::sync::Semaphore`. This eliminates spurious wakeups, race conditions, and latency spikes under heavy concurrent asset processing.
- **Background Rate Limiting:** Added a 50ms artificial delay between requests in the remote cache background migration task. This prevents the application from hammering the backend and triggering automated IP bans for users during heavy startup migrations.
- **General Code Health:** Fixed various `clippy` lints (`needless_continue`, `manual_let_else`) and ran an automated codebase formatting pass.
