# 🚀 ISpooferMotion v2.4.1 (2026-07-14)

## 🐛 Bug Fixes

- **Cookie Validation:** Fixed an issue where cookies auto-detected from Roblox Studio or browsers would fail validation. The cookie sanitizer now strips every character outside the visible ASCII range (`0x20–0x7E`), preventing internal HTTP header rejections caused by non-printable characters embedded by Chrome's DPAPI encryption layer or Studio's credential store.
- **Resilient Auto-Detect:** The auto-detect flow in Credentials settings no longer discards valid cookies on transient network errors (like timeouts or Roblox 429 rate-limits). It now correctly distinguishes genuine auth rejections (HTTP 401/403) from connectivity issues, leaving the existing cookie intact on temporary failures.
- **Memory Injection Restored:** Fixed auto replacement doing nothing in memory injection mode. Memory injection patches asset IDs directly into Studio's process memory and requires exact byte-length matches. Previously, any length mismatch would abort the entire batch. Now, length-mismatched pairs are skipped individually, allowing same-length pairs in the batch to patch successfully.
- **Replacement Pipeline Reliability:** Addressed widespread reports of auto and manual replacements only applying to a handful of assets instead of all mapped assets. Previously, when skipping already-uploaded assets, their mappings from prior runs (`lastReplacements`) were silently discarded, and only newly-uploaded mappings were sent to the Studio plugin. The daemon now merges all accumulated replacement mappings from the session before dispatching, ensuring every known mapping is patched in Studio.

## 🛠 Internal / Technical

- Extracted a `sanitize_cookie_value()` utility in the Rust backend and applied it uniformly across all Roblox API requests.
- Prevented the memory injection engine's `length mismatch` outcome from triggering user-facing error toasts in the UI, as the Studio plugin gracefully handles those unpatched pairs.
