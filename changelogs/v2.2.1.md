## [2.2.1] - 2026-07-10

### ✨ What's New

- **Test Infrastructure Expansion**: Introduced a massive expansion to the test suite, adding E2E UI tests via Playwright, full Vitest environments with mock setups, and a custom lightweight test runner for the Luau plugin.
- **Tauri Plugin Updates**: Upgraded the Tauri app setup to expose the runtime to the browser (`withGlobalTauri: true`) and integrated the Playwright plugin for CDP testing on debug builds.
- **Updater UI Changes**: Updated the updater UI with a cleaner white spinner (replacing indigo) using native Tailwind animations and removed background transparency.

### 🐛 Bug Fixes & Under the Hood

- **Asset ID Substring Corruption**: Fixed a critical bug in the patcher and script rewriter where short asset IDs (e.g., `12345`) could inadvertently corrupt longer IDs (e.g., `123456789`) during string replacement.
- **Studio Connection Stability**: Re-architected `discoverServer` to probe loopback addresses exclusively, removing external DNS dependencies (`nip.io`) to improve reliability during rapid Studio reconnects.
- **Patch Stability**: Anchored asset ID regex matching to prevent greedy extraction from malformed query parameters, and implemented a strict `newTagSet` approach to safely preserve unrelated `CollectionService` tags.
- **Codebase Hardening**: Improved string parsing logic in IPC payloads, added early-exit guard clauses during patching, and secured `ChangeHistoryService` recordings to commit reliably even when individual patches throw errors.
- **Asset Scanning Tweaks**: The Luau scanner now guards against duplicate property polling and properly includes the new `AudioDeviceInput` class when gathering assets.
- **CI/CD Fixes**: Corrected GitHub Actions pipeline corruption caused by Windows/POSIX newline mismatches in Tauri signing keys by introducing a dedicated Node.js normalization step.
- **Build Quality**: Enforced stricter validation pipelines by adding Husky pre-commit hooks and `lint-staged` rules spanning TypeScript, Rust, and Luau files.
- **Performance Optimizations**: Replaced ephemeral HTTP clients with a shared connection pool, eliminating per-request connection overhead. Rewrote file path sanitization with a faster constant-time jump table, greatly improving string processing speed.
- **Async Execution Safety**: Converted synchronous filesystem operations during plugin synchronization and app uninstallation into non-blocking `tokio` equivalents, preventing thread pool deadlocks.
- **Native OS Interactions**: Switched to using `tauri-plugin-opener` for opening system folders natively, moving away from manually spawning fragile shell commands.
- **Macro Cleanup**: Stripped unused Tauri command macro re-exports, reducing codebase noise and improving compilation efficiency.
